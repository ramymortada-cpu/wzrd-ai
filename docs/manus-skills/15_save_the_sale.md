# Skill 15: Save The Sale — handle_cancellation_request

## When to Use
عند طلب العميل إلغاء الطلب — المنطق لمحاولة "إنقاذ البيع" قبل الإلغاء الفعلي.

## اقرأ أولاً
- `radd/pipeline/orchestrator.py` — تدفق المعالجة
- `radd/actions/salla.py` — استعلام الطلبات
- `radd/actions/base.py` — Action dispatcher

## المهمة
كتابة `handle_cancellation_request`:
- يستقبل نية إلغاء من العميل
- يستعلم عن حالة الطلب من Salla/Shopify
- إن كان قابلاً للإلغاء: يعرض خيارات (خصم، تأجيل، استبدال)
- إن لم يكن: يوضح للعميل السبب
- 13 اختبار وحدة

## Acceptance Criteria
- [ ] الدالة تعمل مع Salla و Shopify
- [ ] 13 اختبار يمرون
- [ ] لا تعدّل الطلب فعلياً دون تأكيد العميل
- [ ] الردود بالعربية مع دعم اللهجات

## لا تفعل
- لا تستدعي API إلغاء الطلب دون تأكيد صريح
- لا تكشف تفاصيل الطلب الداخلية للعميل (مثل هامش الربح)
